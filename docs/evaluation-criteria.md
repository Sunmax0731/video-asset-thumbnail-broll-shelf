# 評価基準

## QCDS 定義

- Quality: 実装、テスト、security/privacy、traceability、release evidence の品質。
- Cost: 導入、運用、保守、追加検証に必要な負担。
- Delivery: closed alpha として配布、検証、再実行できる状態。
- Satisfaction: ユーザーが迷わず次アクションに進める UI/UX、docs、手順。

## 評価スケール

`S+ / S- / A+ / A- / B+ / B- / C+ / C- / D+ / D-`

手動テスト未実施の場合、最高評価は原則 `S-`。A- 未満が出た場合は gap analysis に理由と改善を記録する。
