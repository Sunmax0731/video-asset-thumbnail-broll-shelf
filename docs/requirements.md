# 要件定義

## 背景

映像、音声、画像、AEコンポのリンク監査、プロキシ、サムネイル、尺、Bロール候補、使用済み状態を一覧する。

## 課題

素材が増えるとリンク切れ、重複、未使用候補、Bロール探しに時間がかかる。

## 対象ユーザー

- 小規模制作、検品、運用、レビューを少人数で回すユーザー。
- 既存ツールが重い、横断証跡が残らない、導入前の確認手順が弱いと感じているユーザー。
- JSON サンプル、ローカルファイル、レビュー表、導入メモを一つの流れで扱いたいユーザー。

## 機能要件

- FR-1: 入力項目を JSON として取り込み、必須項目を検証できる。
- FR-2: 正常、警告、失敗を同じレポートで確認できる。
- FR-3: 判断理由、証跡、次アクションを保存できる。
- FR-4: 代表シナリオで回帰確認できる。
- FR-5: 導入、ユーザー操作、手動テスト、release evidence を docs として同梱する。

## 非機能要件

- ローカルファーストで動作し、入力ファイルを外部送信しない。
- 小規模チームの代表データで即時に検証できる。
- Markdown / JSON / JS / HTML / Python は UTF-8 + LF を維持する。
- 手動テスト未実施であることを release 前に明示する。

## MVP後の要件

- UI/UX polish、責務分割、security/privacy、traceability、docs ZIP、release asset の再確認を含める。
- QCDS の各観点を A- 以上に維持する。
